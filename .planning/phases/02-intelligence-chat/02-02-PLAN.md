<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NewsAPI response → briefing JSON | External data; article titles and URLs included in briefing without sanitization |
| yfinance OHLCV → indicator computation | External data; float values only, no code execution |
| backend/core → SQLite | All writes parameterized; no f-string SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | NEWSAPI_KEY in config.py | mitigate | Key read from env var via settings.NEWSAPI_KEY — never hardcoded; .env.example has placeholder; .gitignore covers .env |
| T-02-05 | Tampering | news_cache articles_json | mitigate | json.dumps() serialization before INSERT; parameterized ? placeholder — no SQL injection possible |
| T-02-06 | Denial of Service | NewsAPI 100 req/day quota | mitigate | Date-keyed cache in news_cache prevents re-fetching same day's results; cache hit returns immediately without API call |
| T-02-07 | Tampering | yfinance OHLCV data | accept | yfinance data is market data (float prices); no code execution path; indicator computation is pure numeric; low risk |
| T-02-08 | Spoofing | Article URLs in news response | mitigate | URLs from NewsAPI included as-is in article dict; frontend renders as `<a href target="_blank" rel="noopener noreferrer">` — user navigates to external site by intent |
| T-02-SC | Tampering | ta/newsapi-python pip installs | mitigate | Both packages [OK] per RESEARCH.md slopcheck audit; installed in Plan 01 |
</threat_model>
